01第1章　なぜClaude Code × Obsidianで仕事が3倍速になるのか

第1章 なぜ仕事が3倍速になるのか

1-1　仕事が遅い本当の理由

「もっと効率よく仕事したい」と思わない人はいないでしょう。

それなのに、なぜ多くのビジネスパーソンは仕事の効率化に苦しんでいるのでしょうか。

答えは意外なところにあります。

仕事が遅い原因は、能力や経験の問題ではありません。

本当の問題は「情報の流れが断ち切られている」ことにあります。

たとえば、あなたは会議で重要な決定事項をメモしました。

でも、そのメモはスマートフォンのメモアプリに入ったまま、翌週の報告書作成時には「どこに書いたっけ？」と探し回ることになる。

探している時間、実は何も仕事が進んでいません。

あるいは、先週調べた情報が今日また必要になった。

でも、あの資料はどのフォルダに保存したか思い出せず、また同じことを調べ直す。

このような「情報の再探索コスト」が、仕事の時間の30〜40%を占めているという研究があります。

情報の流れが変わると仕事が変わる

さらに、メールへの返信、定型的なレポート作成、データの転記作業。

これらは「考える仕事」ではなく「こなす仕事」です。

でも、一日の中でこういった作業に費やす時間は非常に多い。

Anthropicの調査によれば、Claude Codeを活用したビジネスパーソンは週平均12.3時間の時間削減を達成しています。

週5日で割ると、毎日約2.5時間が手元に戻ってくる計算です。

1-2　Claude Codeとは何か — 「AIアシスタント」の進化版

Claude Codeについて、まず正確に理解しておきましょう。

「ChatGPTと何が違うの？」という疑問をよく聞きます。

ChatGPTをはじめとする一般的なAIチャットは、「質問→回答」の形式で使います。

あなたが文章を打ち込み、AIが答えを返す。

それだけです。

Claude Codeは違います。

Claude Codeは、あなたのパソコン内のファイルに直接アクセスし、操作することができます。

つまり、「この書類をまとめて」「この情報を整理して報告書にして」という指示を受けたとき、実際にファイルを開き、読み込み、新しい文書を生成して保存するところまで、一連の作業を自動で実行してくれます。

Claude Codeと一般AIチャットの違い

エンジニアがコード開発に使うイメージが強いClaude Codeですが、実際には文章・手順・ファイルをまたぐすべての仕事に対応できます。

具体的には：

- 会議のメモから議事録を自動生成し、関係者へのメール文まで作成する

- 複数のExcelファイルを読み込み、月次レポートを一発で生成する

- 大量のメールを分析し、優先度と返信内容を提案する

- 参考資料を読み込んで、プレゼンのスライド構成案を作る

これらを、日本語で話しかけるだけで実行してくれます。

1-3　Obsidianとは何か — 「第二の脳」を作るツール

Obsidianは、ローカル型のノートアプリです。

「ローカル型」というのは、データがあなたのパソコンの中に保存されるということです。

クラウド上にデータが飛ばないため、機密情報も安心して保存できます。

Obsidianの最大の特徴は、ノート同士をリンクで結べることです。

普通のメモアプリは、ノートがバラバラに存在するだけです。

Obsidianでは、「このノートはあのノートと関連している」というリンクを張れます。

これにより、あるアイデアから関連する情報へ、芋づる式にたどっていける「知識のネットワーク」が生まれます。

人間の脳も、記憶は孤立したものではなく、関連する記憶と結びついて存在しています。

Obsidianはまさに、この脳の仕組みを模倣した「第二の脳」なのです。

Obsidianで作る知識ネットワーク

1-4　2つのツールが「組み合わさる」と何が起きるか

Claude CodeとObsidianを単独で使っても、それぞれ便利なツールです。

でも、この2つを連携させると、全く別の次元の生産性が生まれます。

仕組みはシンプルです。

Obsidianに蓄積された「あなたの知識と情報のすべて」を、Claude Codeが参照できるようになる。

つまり、あなたが過去1年間に書いたメモ、蓄積した情報、整理したプロジェクト情報を、すべてClaude Codeが「知っている」状態で仕事を手伝ってくれます。

「先月の○○プロジェクトの議事録から、まだ未解決の課題を抽出して」

「この2年間の顧客メモを分析して、共通する悩みのパターンを教えて」

こういった指示が、実現できるようになります。

「3倍速」という言葉は大げさに聞こえるかもしれません。

でも、実際の内訳を見ると、これは十分にリアルな数字です。

① 情報の再探索がゼロになる（時間削減: 約30%）

蓄積された知識を参照しながらClaude Codeが提案を作るため、「やり直し」が大幅に減ります。

2-4　リンクの張り方 — Obsidianの最強機能

日次ノートに慣れてきたら、次はリンクを使いましょう。

Obsidianでリンクを張る方法は、`[[` を入力するだけです。

リンクを積極的に使うことで、情報の「孤島」がなくなります。

3-3　非エンジニアが陥りがちな3つの誤解

Claude Codeを使い始めた非エンジニアの方がよく陥る誤解があります。

あらかじめ知っておくと、つまずかずに使いこなせます。

誤解1:「プログラミングの知識が必要だ」

不要です。

Claude Codeへの指示はすべて日本語で構いません。

「このファイルを分析して」「こういう文書を作って」という自然な言葉で伝えるだけです。

誤解2:「ミスをしたら取り返しがつかない」

Claude Codeはファイルを削除する操作は、明示的に指示しない限り行いません。

基本的に「読む」「新しく作る」「修正する」の操作がメインです。

また、Windowsなら「元に戻す（Ctrl+Z）」、Macなら「Command+Z」でいつでも取り消せます。

誤解3:「一回の指示で完璧な結果が出るはず」

Claude Codeとの作業は、会話のキャッチボールです。

最初の出力に「もう少しこうして」「この部分をもっと詳しく」と追加指示することで、どんどん良くなっていきます。

最初から100点を求めず、60点の出力から対話で100点に近づけていくイメージを持ちましょう。

Claude Codeへの正しい使い方

3-4　仕事別・実践的な指示フレーズ集

Claude Codeに指示するときは、以下のような「フレーズのパターン」を使うと効果的です。

情報整理・分析系

- 「〇〇のファイルを読んで、重要なポイントを3〜5点にまとめてください」

- 「この資料の中から、□□に関連する部分だけ抜き出してください」

- 「先月の日次ノートを分析して、時間をとられているタスクのパターンを教えてください」

文書作成・加工系

- 「この箇条書きメモを、ビジネスメールの文体に整えてください」

- 「この報告書の下書きを、上司が読みやすいように構成を整理してください」

- 「以下の素材から、1000字程度の提案書の冒頭文を書いてください」

4-1　MCP連携とは何か — 「つなぐ橋」を架ける

MCP（Model Context Protocol）は、Claude CodeとObsidianを直接つなぐための「橋」です。

難しそうな名前ですが、要はObsidianのデータをClaude Codeがリアルタイムで読み書きできるようにする仕組みです。

MCPを設定すると、こんなことが可能になります：

- Obsidianに新しいノートを自動で追加する

- Claude Codeとの会話内容を自動的にObsidianに保存する

- 「今日の日次ノートに〇〇を追記して」という指示が使えるようになる
